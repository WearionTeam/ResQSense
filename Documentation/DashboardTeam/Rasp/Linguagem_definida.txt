

The biomedical monitoring system adopts a dual stack architecture to capitalize on the specialties of each component. The vest module, centered on the ESP32 microcontroller, will be programmed in C (representing the first stack: embedded data acquisition). This choice is justified by the need for maximum efficiency, low power consumption, and real-time performance for accurate reading of I²C sensors, such as the MPU6050. In contrast, the visualization interface, running on the Raspberry Pi 3, will be developed in Python (representing the second stack: data visualization and management). Python is the most suitable language for the Raspberry Pi, providing robust libraries (such as PyQT or Kivy) for creating the graphical user interface (GUI) and visual alert logic. Additionally, Python offers simple drivers for the database (e.g., SQLite), ensuring the persistence of biomedical data for historical analysis, which optimizes clarity and speed of development on the team leader's side.
